# How fast actually GO is?

### Go is consider *fast* in a few different ways - we often mix them together

1. Fast Execution/runtime performance.
2. Fast Compilation/build time
3. Fast Concurrency handling
4. Fast Development Productivity

## 1. Runtime Speed (Execution Speed)
This means *"How fast does the program run after it is compiled?"*

Go is compiled language that compiles directly to machine code which makes it faster than any interpreted languages like ***python*** and ***javascript*** but usually slower than ***C***, ***C++*** and ***Rust*** because ***Go*** has


* Garbage Collection
* Runtime Scheduler
* Safety overhead

## 2. Compilation Speed (Converting to Executable)
 This means *"How fast source code becomes executables?"*. 

It compiles the code very fast compared to ***C***, ***C++*** and ***Rust*** becuase ***Go*** has


* Simple Language Design
* Small Syntax
* Fewer compiler complexities
* Simpler dependency management

## 3. Concurrency Performance

Go is extremely efficient for:

* APIs
* servers
* networking
* distributed systems
* microservices

because of:

* goroutines
* channels
* lightweight scheduler
